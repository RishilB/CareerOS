# Learning Guide

## Purpose

This guide defines how CareerOS should help students identify skill gaps and create learning roadmaps aligned to career goals.

## Objectives

CareerOS should help students:

- Identify skills required for target roles.
- Prioritize learning by career impact.
- Build practical learning plans.
- Convert learning into evidence.
- Avoid random content consumption.
- Use AI tools effectively for career growth.

## Learning Philosophy

Learning is useful when it changes capability and creates evidence.

CareerOS should not recommend endless courses. It should help students build skills, projects, artifacts, and confidence that improve employability.

## Skill Gap Framework

Use the GAP framework:

- Goal: target role or capability
- Ability: current skills and evidence
- Plan: specific learning actions and outputs

## Skill Categories

### Functional Skills

Role-specific business skills.

Examples:

- Marketing: segmentation, consumer research, campaign metrics
- Finance: financial statements, valuation, ratio analysis
- HR: talent acquisition, engagement, performance management
- Operations: process mapping, supply chain, quality improvement
- Analytics: SQL, dashboards, data storytelling
- Consulting: issue trees, hypothesis thinking, market sizing

### Technical Tools

Tools relevant to roles.

Examples:

- Excel
- Power BI
- Tableau
- SQL
- Python
- Canva
- CRM tools
- analytics platforms

### Communication Skills

Skills that convert knowledge into influence.

Examples:

- presentation
- email writing
- storytelling
- executive summaries
- interview answers
- group discussion

### Career Execution Skills

Examples:

- networking
- application tracking
- follow-up
- negotiation
- weekly planning

## Learning Roadmap Structure

A strong roadmap includes:

- Target role
- current level
- priority skills
- weekly schedule
- learning resources
- practice tasks
- output artifacts
- feedback loop
- success criteria

## Evidence-Based Learning

Every plan should produce evidence:

- Case analysis
- dashboard
- report
- project summary
- presentation
- article
- portfolio page
- interview story
- resume bullet

Learning without evidence is less useful for placements.

## Prioritization

Prioritize skills using:

- Relevance to target role
- gap severity
- time required
- proof potential
- interview importance
- student interest

## Common Mistakes

- Learning without a target role.
- Taking courses without producing artifacts.
- Trying to learn too many tools at once.
- Ignoring communication.
- Not practicing with real business cases.
- Not connecting learning to resume or LinkedIn.

## Learning Checklist

- Is the target role clear?
- Are top 3 skills identified?
- Is the plan realistic for available time?
- Does each week produce output?
- Can the output become resume evidence?
- Is there a feedback loop?
- Is the learning plan connected to interviews?

## Expected Outputs

CareerOS may produce:

- Skill gap analysis
- 7-day plan
- 30-day roadmap
- 90-day roadmap
- project ideas
- resource plan
- learning tracker
- resume bullet from learning project

## Decision Tree

1. Is the target role clear?
   - Yes: map role skills.
   - No: run career discovery first.
2. Does the student have time constraints?
   - Yes: build a realistic plan.
   - No: create a deeper roadmap.
3. Does the student need proof for applications?
   - Yes: prioritize portfolio outputs.
   - No: prioritize conceptual mastery and practice.
